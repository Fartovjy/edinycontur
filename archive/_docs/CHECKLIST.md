# CHECKLIST MVP «Единый Контур»

Ручной чек-лист для проверки основных сценариев перед демонстрацией или передачей версии.

## Подготовка

- [ ] Проект запускается командой `docker compose up --build`.
- [ ] Открывается [http://localhost:8000](http://localhost:8000).
- [ ] Открывается админка [http://localhost:8000/admin/](http://localhost:8000/admin/).
- [ ] Демоданные загружены командой `python manage.py seed_demo_data`.

## Вход Под Разными Ролями

- [ ] `admin / admin` входит в систему и видит список заявок, dashboard и админку.
- [ ] `operator / password` входит в систему и видит список заявок.
- [ ] `supply / password` входит в систему и видит список заявок.
- [ ] `transport / password` входит в систему и видит список заявок.
- [ ] `warehouse / password` входит в систему и видит список заявок.
- [ ] `driver / password` входит в систему и видит свои заявки.
- [ ] `manager / password` входит в систему и видит dashboard.

## Создание Заявки

- [ ] Войти под `operator`.
- [ ] Открыть создание заявки.
- [ ] Заполнить клиента, адрес, контакт, регион, склад, груз, вес, объём, даты и ЧЗ.
- [ ] Сохранить заявку.
- [ ] Проверить, что заявка создана и получила номер формата `EK-YYYY-NNNNN`.

## Смена Статусов

- [ ] Открыть карточку заявки.
- [ ] Перейти к редактированию под ролью, которой доступна смена нужного статуса.
- [ ] Выполнить допустимый переход статуса.
- [ ] Проверить, что статус изменился.
- [ ] Проверить, что в истории статусов появилась запись со старым статусом, новым статусом, пользователем, временем и комментарием.
- [ ] Попробовать недопустимый переход и убедиться, что он заблокирован.

## Загрузка Файла

- [ ] Открыть карточку заявки.
- [ ] Загрузить PDF-файл.
- [ ] Загрузить JPG/PNG/WEBP-фото.
- [ ] Проверить, что вложения появились в карточке заявки.
- [ ] Проверить отображение типа файла, имени, пользователя, даты загрузки и ссылки на просмотр/скачивание.
- [ ] Попробовать загрузить файл запрещённого типа и убедиться, что форма показывает ошибку.

## Создание Проблемы

- [ ] Открыть карточку заявки.
- [ ] Нажать `Сообщить о проблеме`.
- [ ] Заполнить тип проблемы, описание, ответственного.
- [ ] При необходимости приложить файл/фото.
- [ ] Сохранить проблему.
- [ ] Проверить, что заявка перешла в статус `problem`.
- [ ] Проверить, что проблема появилась в карточке заявки.
- [ ] Проверить, что в истории статусов появилась запись о переходе в `problem`.

## Закрытие Проблемы

- [ ] Открыть проблемную заявку.
- [ ] В блоке проблем заполнить комментарий решения.
- [ ] Выбрать новый рабочий статус или `closed`.
- [ ] Закрыть проблему.
- [ ] Проверить, что `ProblemReport.status` стал `resolved`.
- [ ] Проверить, что заполнено время решения.
- [ ] Проверить, что заявка переведена в выбранный статус.

## Назначение Транспорта

- [ ] Войти под `transport`.
- [ ] Открыть заявку без транспорта.
- [ ] Назначить машину.
- [ ] Назначить водителя.
- [ ] При необходимости указать дату отправки.
- [ ] Перевести заявку в статус `transport_assigned`.
- [ ] Проверить, что машина и водитель отображаются в списке и карточке заявки.

## Отметка Доставки

- [ ] Войти под `driver`.
- [ ] Открыть свою заявку.
- [ ] Перевести заявку в статус `in_transit`, если она ещё не в пути.
- [ ] Отметить `Доставлено`.
- [ ] Проверить, что статус стал `delivered`.
- [ ] Проверить, что заполнена фактическая дата доставки.
- [ ] Проверить, что запись появилась в истории статусов.

## Проверка Dashboard

- [ ] Войти под `manager` или `admin`.
- [ ] Открыть dashboard.
- [ ] Проверить показатели: активные, просроченные, проблемные, без транспорта, ожидают склад, в пути, доставлены сегодня.
- [ ] Проверить списки: топ просроченных, открытые проблемы, заявки без транспорта, доставки на сегодня, доставки на завтра.
- [ ] Сравнить несколько значений со списком заявок и быстрыми вкладками.

## Проверка Telegram-Уведомления

- [ ] Указать `TELEGRAM_BOT_TOKEN` и `WEB_APP_BASE_URL` в `.env`.
- [ ] Запустить web и bot через `docker compose up --build`.
- [ ] Заполнить `telegram_id` или `telegram_chat_id` тестового пользователя/водителя.
- [ ] Отправить боту `/start`.
- [ ] Проверить, что бот показывает роль и доступные команды.
- [ ] Сменить статус заявки и проверить уведомление ответственной роли.
- [ ] Для водителя проверить кнопку `Доставлено`.
- [ ] Для водителя проверить кнопку `Проблема`.
- [ ] Проверить, что ссылка из Telegram открывает карточку заявки.
